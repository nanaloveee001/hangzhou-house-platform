# 高德地图 API Key 配置指南

> 依据规格书 2.3。Key 仅用于**服务端爬虫**（GitHub Actions Secrets / 本地环境变量），**不进入前端**，满足 7.1 安全要求。

## 步骤 1：注册开发者

1. 打开 [高德开放平台](https://lbs.amap.com/) → 右上角「注册」。
2. 使用**本人手机号**完成注册，并按提示完成**实名认证**（个人/企业均可）。
   - 此步骤必须由你本人完成，无法由他人代劳（涉及账号归属与配额）。

## 步骤 2：创建应用与 Key

1. 进入「控制台 → 应用管理 → 我的应用 → 创建新应用」。
2. 应用名称随意（如 `hangzhou-house`），类型选「其他」。
3. 在应用下点击「添加 Key」：
   - **服务平台**：选择 **Web 服务**（用于爬虫的服务端 POI / 地理编码 API）。
   - 填写 Key 名称，提交后获得 `Key` 值。

## 步骤 3：配置到本项目

### 本地运行
```bash
export AMAP_KEY="你刚刚获得的Key"
python crawler/crawler.py
```

### GitHub Actions（推荐，自动每日抓取）
仓库 **Settings → Secrets → Actions → New repository secret**：
- Name: `AMAP_KEY`
- Secret: 你的 Key

工作流会读取该 Secret 运行爬虫，Key 不会出现在任何代码或前端中。

## 步骤 4：配额与合规

- 高德 Web 服务免费配额通常为 **每日 3000 次**（以官网为准），日常学区/配套抓取足够。
- 严格遵守《高德地图 API 使用条款》与 `robots.txt`；本项目**不抓取链家/贝壳等禁止源**。
- 若需更高配额，按官网指引升级，并在产品中标注。

## 备注：为什么前端不用高德 JS API Key？

本方案采用 **Leaflet + OpenStreetMap** 作为前端底图（keyless、合规、真实开放），所有高德数据由**服务端爬虫预取为静态 JSON** 后前端直接渲染。这样：
1. 前端零 Key，彻底避免 Key 暴露（7.1）；
2. 地图底图为真实开放数据；
3. 中国学区/POI 精度由高德服务端保证。
